## Monitoring MongoDB Performance using Mongotop
This guide demonstrates how to monitor MongoDB performance using **mongotop**, showing **read/write** activity across collections in **real time**. It includes benchmark results collected on Azure Arm64 virtual machines, providing a reference for expected latencies.

## Run mongotop â€” Terminal 2

```console
mongotop 2
```
**mongotop** shows how much time the server spends reading and writing each collection (refreshes every 2 seconds here). It helps you see which collections are busiest and whether reads or writes dominate.

## Explanation of Metrics and Namespaces

**Metrics**

  - **ns (Namespace)** â€“ Identifies the specific database and collection being measured.
  - **total** â€“ Total time spent on both read and write operations.
  - **read** â€“ Time taken by read operations like queries or fetches.
  - **write** â€“ Time taken by write operations like inserts, updates, or deletes.
  - **timestamp** â€“ Marks when the metric snapshot was captured.

**Namespaces**

  - **benchmarkDB.testCollection** â€“ Core benchmark collection with balanced read/write load.
  - **admin.atlascli** â€“ Tracks admin-level client activity.
  - **benchmarkDB.cursorTest** â€“ Measures cursor operations during benchmarking.
  - **config.system_sessions_bench** â€“ Benchmarks session handling in config DB.
  - **config.transactions_bench** â€“ Evaluates transaction performance in config DB.
  - **local.system_replset_bench** â€“ Tests replication set metadata access.
  - **test.admin_system_version_test** â€“ Monitors versioning metadata in test DB.
  - **test.atlascli** â€“ Simulates client-side workload in test DB.
  - **test.system_sessions_bench** â€“ Benchmarks session handling in test DB.
  - **admin.system.version** â€“ Static metadata collection with minimal activity.
